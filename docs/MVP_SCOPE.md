# MVP: границы первой версии

Цель MVP — доказать, что топология с границами работает и удобна, а не собрать максимум фич. «Сначала границы, потом удобства».

## Входит

- **Один канал:** Telegram (официальный Bot API), deny-by-default авторизация, pairing владельца.
- **Host-ядро:** входная/выходная очереди (SQLite), петля оркестратора, gate engine, audit log.
- **Sandbox:** один бэкенд (Docker), deny-all egress, allowlist mount, non-root.
- **Broker:** локальный proxy — агент не видит ключи (вынос на отдельный хост — позже).
- **Память:** эпизодическая (полнотекстовый поиск) + семантическая со статусами; курация только детерминированная (без LLM-консолидации в MVP).
- **Навыки:** декларативные (markdown + манифест); код-навыки — минимально, только через sandbox.
- **Карантин:** минимальная версия — Q-LLM для пересланного контента и веб-страниц.
- **Бюджет:** дневной лимит, приоритет интерактива, явная деградация.
- **Градуированная цена действия:** read-only / reversible / irreversible с гейтами.

## НЕ входит (осознанно отложено)

- Другие каналы (WhatsApp, Discord, Slack, email как канал) — по одному и позже, только официальные API.
- LLM-консолидация памяти (в MVP — только детерминированная курация).
- Мультимодельные конфигурации (P-LLM/Q-LLM разных провайдеров).
- 2FA/out-of-band подтверждение необратимых действий (в MVP они ограничены и требуют явного подтверждения в чате).
- Вынос broker на отдельный хост / micro-VM для sandbox.
- Marketplace навыков любого вида.
- Multi-tenant / командные сценарии.

## Критерии готовности MVP

- [x] Prompt injection из пересланного письма не приводит к вызову инструмента (тест V1 из THREAT_MODEL).  
      **Proof:** `test/security/v1-prompt-injection.test.ts`
- [x] Агент не имеет доступа к сырому секрету ни в одной точке (тест V2).  
      **Proof:** `test/security/v2-secret-exfiltration.test.ts`
- [x] Код навыка не может достать файлы хоста или выйти в сеть мимо broker (тест V3).  
      **Proof:** `test/security/v3-sandbox-escape.test.ts`
- [x] Знание из недоверенного источника не попадает в системный контекст без промоушена (тест V4).  
      **Proof:** `test/security/v4-memory-poisoning.test.ts`
- [x] Воспроизводимое знание автоматически становится corroborated и переиспользуется (проверка «не тупой»).  
      **Proof:** `test/integration/promotion-loop.test.ts`, `test/unit/verifier.test.ts`; метрика `reuse_rate` — `/metrics`, `src/memory/metrics.ts`
- [x] Исчерпание бюджета даёт уведомление + деградацию, а не тихий сбой (тест V7).  
      **Proof:** `test/integration/budget-loop.test.ts`
- [x] Ядро укладывается в целевой порядок ~4K LOC и читается за разумное время.  
      **Proof:** `npm run loc` (`scripts/loc.mjs`, порог 4000)

## Метрика успеха MVP

Владелец может доверить агенту чтение недоверенного контента (почта, ссылки) и рутинные обратимые действия 24/7, **не опасаясь**, что инъекция уведёт секрет или отравит память, — и при этом агент действительно помогает (не «безопасный, но бесполезный»).
