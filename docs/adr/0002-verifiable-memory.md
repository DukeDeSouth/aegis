# 0002. Память с эпистемическими статусами

- **Статус:** Accepted
- **Дата:** 2026-07-05

## Контекст

Hermes накапливает память и самозаписываемые навыки без проверки истинности. Это дало новый вектор — self-poisoning: одна скомпрометированная сессия отравляет все будущие. Одновременно отсутствие обучения вообще (NanoClaw) лишает агента главной ценности — становиться полезнее со временем.

## Решение

Каждая запись семантической памяти несёт `epistemic_status` (`unverified`/`corroborated`/`verified`/`refuted`), `provenance` (источник/домен) и `evidence[]`. В системный контекст будущих сессий попадает только знание, прошедшее promotion-гейт:

- `corroborated` (прошло детерминированную проверку) — автоматически;
- `verified` — требуется для необратимых/дорогих действий.

Курация двухфазная: детерминированная всегда, LLM-консолидация по бюджету. Snapshot/rollback перед мутациями.

## Последствия

**Плюсы:** self-poisoning закрыт (unverified не влияет на будущее); обучение сохранено; воспроизводимое проходит без человека (не «тупой» агент).

**Минусы:** риск переверификации (если планка слишком высока) и раздувание evidence. Митигация — градация corroborated/verified; evidence несёт provenance и не самоподтверждается из карантина.

## Альтернативы

- Память без статусов (Hermes) — отвергнута (self-poisoning).
- Только ручной approve всего — отвергнут (убивает автономность).
