# 0003. Навыки как данные по умолчанию

- **Статус:** Accepted
- **Дата:** 2026-07-05

## Контекст

Marketplace исполняемого кода — доказанная supply-chain-дыра: кампания ClawHavoc распространила 1184 вредоносных пакета для агентов класса. Установка через `curl|bash` и runtime-подгрузка кода в хост-процесс усугубляют риск.

## Решение

Навык по умолчанию — данные: markdown-процедура + JSON-манифест capabilities, без исполняемых файлов. Код разрешён, но только:

- исполняется в sandbox (не в хост-ядре);
- проходит verifiable loop (unverified → dry-run → corroborated);
- `agent-created` сканируется всегда;
- заявляет capability-манифест; устанавливается из git с pinned-версией.

Запрещены `curl|bash` и runtime-install пакетов в хост-процесс.

## Последствия

**Плюсы:** payload негде спрятать в декларативном навыке; код изолирован и проверяется; supply-chain-поверхность резко сужена.

**Минусы:** ~20% задач требуют кода и это самые ценные — их путь длиннее (sandbox + проверка). Принято: это не запрет кода, а его изоляция и верификация.

## Альтернативы

- Marketplace исполняемого кода (OpenClaw ClawHub) — отвергнут.
- Полный запрет кода — отвергнут (теряется выразительность).
