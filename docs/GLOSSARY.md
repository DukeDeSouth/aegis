# Глоссарий

**Lethal trifecta** — сочетание в одном контексте трёх факторов: чтение недоверенного контента, доступ к секретам, способность действовать. Термин Unit 42 (Palo Alto). Именно оно превращает prompt injection в удалённое выполнение кода.

**Trust-домен** — изолированная зона исполнения с границей на уровне ОС (контейнер, отдельный процесс). В Aegis их четыре: Quarantine, Sandbox, Broker, Host.

**Quarantine / Q-LLM** — домен и модель, обрабатывающие недоверенный контент без доступа к инструментам. Недоверенные данные могут войти в рассуждение, но не могут инициировать действие в том же ходе. Публичный аналог — CaMeL (Google DeepMind).

**Orchestrator / P-LLM** — привилегированный LLM в доверенном ядре; видит команды владельца (control flow) и результаты, но не сырой недоверенный контент.

**Sandbox** — контейнер, где исполняется любой код/shell; deny-all egress, allowlist mount, non-root, нет доступа к состоянию хоста.

**Credential Broker** — отдельный процесс, подставляющий секреты на уровне прокси; агент никогда не держит реальный ключ. Аналоги — IETF CB4A, CapSeal, Envoy-sidecar.

**Provenance** — происхождение записи/данных: `owner` / `orchestrator` / `quarantine` / `background`. Влияет на доверие и полномочия.

**Epistemic status** — статус знания: `unverified` → `corroborated` → `verified`, либо `refuted`. Определяет, попадает ли знание в системный контекст.

**Promotion-гейт** — правило допуска знания в системный контекст будущих сессий: `corroborated` — автоматически, `verified` — требуется для необратимых действий.

**Self-poisoning** — атака, при которой скомпрометированная сессия записывает вредное знание/навык, а последующие сессии загружают его как доверенное. Закрывается эпистемическими статусами + promotion-гейтом.

**Evidence** — доказательства, подтверждающие знание (успешный тест, воспроизведённое наблюдение, подтверждение владельца).

**Градуированная цена действия** — классификация действий (read-only / обратимое / необратимое) с разными гейтами, вместо глобального «yolo».

**Fail-closed** — при недоступности проверки/broker/карантина действие отклоняется, а не пропускается.

**reuse_rate** — доля сохранённых знаний, реально переиспользованных; метрика ценности обучения.

**Навык как данные** — навык в виде markdown-процедуры + capability-манифеста, без исполняемого кода в дефолте.

**Capability-манифест** — декларация навыка о том, что ему нужно (инструменты, сеть, класс действия, требуется ли код).

## Референс-проекты (для сравнения)

**OpenClaw** — монолитный агент-фреймворк, hub-and-spoke, скилы = код из marketplace; всё в одном trust-домене.

**Hermes** — модульный агент с накопительной памятью и самозаписью навыков, но без верификации сохраняемого; permissive-дефолты.

**NanoClaw** — минималистичный агент (~500 LOC ядра): исполнение в контейнерах, секреты в vault. Без карантина входа и без верифицируемой памяти — то, что добавляет Aegis.

**CaMeL** — исследовательский фреймворк Google DeepMind: разделение control/data flow против prompt injection. Основа для домена Quarantine.
