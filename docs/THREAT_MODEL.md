# Модель угроз

Формат близок к STRIDE/по-активам. Живой документ; каждый вектор ссылается на митигацию в архитектуре или модели безопасности.

## Активы

| Актив                                       | Почему ценен                           |
| ------------------------------------------- | -------------------------------------- |
| Секреты владельца (API-ключи, SSH, токены)  | доступ к внешним системам, деньги      |
| Приватные данные (переписка, файлы, память) | конфиденциальность                     |
| Способность действовать от лица владельца   | отправка сообщений, оплаты, изменения  |
| Целостность памяти/навыков                  | искажение влияет на все будущие сессии |
| Токен-бюджет                                | деньги, доступность                    |

## Злоумышленники

1. **Внешний через контент** — прячет инструкции в письме/веб-странице/файле, который агент прочитает (prompt injection). Не имеет прямого доступа к машине.
2. **Вредоносный навык/зависимость** — код в навыке или пакете с payload (сценарий ClawHavoc: 1184 вредоносных пакета в marketplace класса).
3. **Захват канала** — компрометация аккаунта владельца в мессенджере.
4. **Локальный/скомпрометированный процесс** — если один из доменов пробит, что он может достать из соседних.

## Вектора и митигации

### V1. Prompt injection во входящем контенте

Инъекция велит выполнить команду и/или выслать секрет.

- **Митигация:** карантин (Q-LLM без инструментов) — недоверенные данные не инициируют tool-call в том же ходе (TRUST_DOMAINS §1). Даже при обходе: секрета у агента нет (broker), необратимое действие требует human-gate (SECURITY_MODEL).
- **Остаточный риск:** оркестратор действует по недоверенным данным напрямую → правило «данные не инициируют действие» обязательно к тестированию.

### V2. Эксфильтрация секретов

Заставить агент прочитать и отправить ключ.

- **Митигация:** broker — агент никогда не видит реальный ключ (TRUST_DOMAINS §3); sandbox имеет deny-all egress, трафик только через broker-proxy.
- **Остаточный риск (local broker):** broker и агент на общем kernel — уязвимость ядра снимает границу; **S1 remote broker** (ADR-0027) выносит секреты на отдельный хост + mTLS.
- **Остаточный риск (remote broker):** компрометация broker-хоста; компрометация core даёт client cert (impersonation), но не secret files.

### V3. Выход исполнения за пределы sandbox

Навык/код пытается достать файлы хоста или сеть.

- **Митигация:** контейнер с границей ОС, non-root, allowlist mount, deny-all egress (TRUST_DOMAINS §2). Состояние хоста недоступно.
- **Остаточный риск (docker runtime):** escape shared-kernel runc — для Linux prod: **gVisor runsc** opt-in (`sandbox.runtime: gvisor`, ADR-0028, `deploy/gvisor/`).
- **Остаточный риск (gvisor):** syscall/emulation gaps — micro-VM (Firecracker/Kata) для максимальной изоляции (Sprint 41+).

### V4. Self-poisoning памяти

Скомпрометированная/обманутая сессия записывает вредное знание/навык, будущие сессии грузят как доверенное.

- **Митигация:** знание рождается `unverified` с provenance; в системный контекст только через promotion-гейт (corroborated/verified); provenance «карантин/фон» понижает доверие; snapshot/rollback (LEARNING_LOOP). Команды записи памяти (`/remember`) — **owner_only** (Sprint 41): scheduler/cron не может подменить provenance на `owner`.
- **Остаточный риск:** отравление слоя доказательств (evidence) — evidence тоже несёт provenance и не самоподтверждается из карантина.

### V8. Компрометация `aegis.config.json` / cron-хинтов

Атакующий с правом записи конфига (или подмена через backup/CI) вставляет вредоносный `schedules[].text`.

- **Что доступно scheduler-провенансу через gate:** обратимые действия — `sandbox.run`, `file.write` (workspace), `message.send`, `llm.invoke`; фоновые read-only (`/search`). **Недоступно:** `/remember`, `/corroborate`, `/verify`, irreversible без human-gate, promotion owner-команд.
- **Митигация:** gate engine разделяет `TRUSTED_FOR_EFFECT` (owner) и `TRUSTED_FOR_BACKGROUND` (scheduler); критичные команды проверяют `provenance === 'owner'` до side-effect; audit `knowledge.denied` / `promotion.denied`.
- **Рекомендации владельцу:** `chmod 0600 aegis.config.json`; review cron-хинтов в dashboard; не хранить конфиг в world-readable volume; отдельный пользователь ОС для процесса агента.
- **Остаточный риск:** легитимный cron с вредоносным текстом всё ещё может тратить токены и писать в workspace — budget engine + мониторинг audit.

### V9. Brute-force pairing / поверхность WebChat

Перебор pairing-кода на `POST /api/pair` или XSS через отсутствие CSP.

- **Митигация (pairing):** счётчик неудач в `channel_state`, lockout после 5 fails, экспоненциальный backoff 60s→15m, audit `pairing.lockout`, 429 до истечения lockout (без `timingSafeEqual` — защита от timing oracle при lockout). Sprint 41.
- **Митигация (CSP):** `Content-Security-Policy`, `X-Content-Type-Options`, `Referrer-Policy` на всех ответах WebChat.
- **Связь с V6:** 2FA на irreversible (`gate.second_factor`) — отдельный контур; pairing lockout закрывает offline перебор кода до захвата сессии.
- **Остаточный риск:** физический доступ к экрану с pairing-кодом; слабый код владельца — рекомендация ≥16 символов из env.

### V5. Вредоносный навык/supply-chain

Payload в навыке или зависимости.

- **Митигация:** навыки как данные (markdown + манифест) — негде спрятать исполняемый payload; код-навыки только в sandbox с capability-манифестом; установка из git с pinned-версией, без `curl|bash` и runtime-install в хосте; agent-created код сканируется всегда (SKILLS_MODEL).
- **Остаточный риск:** легитимный код-навык с логической закладкой → review + verifiable loop (dry-run).

### V6. Захват аккаунта владельца (V3-канал)

Атакующий пишет агенту как владелец.

- **Митигация:** провенанс + для необратимых действий — out-of-band 2FA (`gate.second_factor`: cross-channel TG↔Discord и/или TOTP; дефолт выключен); security-контур V9 (2FA + pairing lockout).
- **Остаточный риск:** при одном paired-канале и без TOTP — только human-gate `/approve` в том же канале (как Sprint 4–26).

### V7. DoS/перерасход токенов

Флуд задачами/контентом сжигает бюджет.

- **Митигация:** budget engine с дневным лимитом; приоритет интерактива владельца над фоном; явная деградация в детерминированный режим (TOKEN_ECONOMY).
- **Остаточный риск:** бюджет исчерпан легитимным фоном до пробуждения владельца → фон под нож первым, уведомление.

## Вне области (MVP)

- Multi-tenant/командные сценарии — single-tenant по дизайну.
- Физический доступ к машине владельца.
- Атаки на самого LLM-провайдера.
- Юридические аспекты ToS мессенджеров (риск, не архитектура; только официальные Bot API в MVP).
