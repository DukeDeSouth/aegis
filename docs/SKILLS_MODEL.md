# Модель навыков

Навыки — способ расширять агента. В Aegis они устроены так, чтобы расширение не открывало supply-chain-дыру (главный урок ClawHavoc: marketplace исполняемого кода = 1184 вредоносных пакета).

## Принцип: навыки как данные

Навык по умолчанию — это **данные, а не код**:

- markdown-процедура (что делать, шаги, когда применять);
- JSON-манифест capabilities (что навыку нужно).

Без исполняемых файлов в дефолте payload негде спрятать. Это покрывает большинство процедурных навыков (инструкции агенту «как сделать X»).

## Capability-манифест

Каждый навык объявляет, что ему нужно:

```json
{
  "name": "digest-inbox",
  "description": "Собрать утренний дайджест почты",
  "needs": ["read_email"],
  "network": "none",
  "action_class": "read-only",
  "code": false
}
```

- `needs` — запрашиваемые инструменты/ресурсы (deny-by-default: не запросил → не получил).
- `network` — `none` | список разрешённых хостов (через broker-proxy).
- `action_class` — `read-only` | `reversible` | `irreversible` (определяет гейт, см. SECURITY_MODEL).
- `code` — требует ли навык исполнения кода.

## Навыки с кодом

Когда данных недостаточно (парсинг, трансформация) — код разрешён, но:

- исполняется **только в sandbox** (никогда в хост-ядре);
- проходит тот же verifiable loop, что и знания: сгенерированный/установленный код = `unverified` → dry-run в изоляции на тестовых данных → `corroborated` → доступен (см. LEARNING_LOOP);
- `agent-created` код **сканируется всегда** (в отличие от дефолта Hermes, где самозаписанные навыки не сканируются);
- заявляет capability-манифест; сеть — только через broker.

Так «агент сам себе написал навык» сохраняется как возможность, но код не касается доверенного ядра и не считается доверенным, пока не проверен в изоляции.

## Установка навыков

- Из git-репозитория с **pinned-версией** (конкретный commit/tag).
- **Запрещено:** `curl | bash`, runtime-установка пакетов в хост-процесс, подгрузка кода из произвольного marketplace.
- Источник несёт trust-уровень; сторонний навык требует review перед допуском к classes выше `read-only`.

## Прогрессивное раскрытие

Агент видит список навыков (имя + краткое описание), а полную процедуру подгружает только при применении — экономит контекст и токены (паттерн agentskills.io / Hermes).

## Открытые вопросы

- Формат и версионирование манифеста.
- Набор стандартных capability (какой гранулярности).
- Политика review для сторонних навыков (ручной / автоматический скан / оба).
